How to do Universal Design...

Many simple adjustments can be easily implemented into your websites to make them universally accessible. Keep reading to discover some of the most important changes you can make, along with code examples.

Semantic HTML

Many developers are taught to simply put different content into various div elements. However we should avoid the overuse of div elements, since this provides no further information to the user about what the purpose of the element is. Using semantic HTML, which includes tags such as header, nav, main, aside, figure, and more, gives the user much more context about what the function of the element is. The code sample below provides an example of using semantic HTML for a basic webpage:

 
                    
    <header> The Main Title for the Page </header>
    <nav> 
        <ul>
            <li><a href = "#section-1">Section 1</a></li>
            <li><a href = "#section-2">Section 2</a></li>
        </ul>
    </nav>
    <main> 
        <section id = "section-1">
            <figure>
                <img src = "img_source.jpg" alt="Description of image.">
            </figure>
        </section>
        <section id = "section-2">
            <p>
                This is all about the second topic...
            </p>
        </section>
    </main>

Skip Navigation

It can be irritating if a user must listen to the entire navigation menu contents everytime they switch to a new page. To solve this problem, we can use a "Skip Navigation" button that pops up before the naviagation, which moves the user down the page right to the main content when clicked.

Below is example HTML code for creating a skip navigation button, to move the user down to the main site content:

 
                     
    <a class= "skip-link" href="#main-content">Skip to main content.</a>
    <header> The Main Title for the Page </header>
    <nav> 
        <ul>
            <li><a href = "#section-1">Section 1</a></li>
            <li><a href = "#section-2">Section 2</a></li>
        </ul>
    </nav>
    <main id="main-content"> 
        <section id = "section-1">
            <figure>
                <img src = "img_source.jpg" alt="Description of image.">
            </figure>
        </section>
        <section id = "section-2">
            <p>
                This is all about the second topic...
            </p>
        </section>
    </main>

Below is example CSS code for creating a skip navigation button, which moves it in and out of sight:

 
                    
    .skip-link{
        background: white;
        color: #000;
        font-weight: 700px;
        padding: 4px;
        position: absolute;
        transform: translateY(-200%);
    }
    
    .skip-link:focus{
        transform: translateY(0%);
    }
    
    .skip-link{
        transition: transform 0.3s;
    }

Screen-reader Only

Elements should never remain untitled, however, there are elements that could be clear to sighted users but perhaps be confusing for those using a screen-reader. One example is the main site navigation. We can provide a label for screen-reader users without making it visible by using a screen-reader only class with CSS.

Below is example HTML code for creating a screen-reader only title for the main navigation:

 
                    
    <nav> 
        <h2 class="sr-only">Main Navigation</h2>
        <ul>
            <li><a href = "#section-1">Section 1</a></li>
            <li><a href = "#section-2">Section 2</a></li>
        </ul>
    </nav>

Below is example CSS code for the screen-reader only class:

 
                    
    .sr-only{
        border: 0;
        clip: rect(0,0,0,0);
        height: 1px;
        margin: -1px;
        overflow: hidden;
        padding:0;
        position: absolute;
        white-space: nowrap;
        width: 1px;
    }

Accessible Forms

Forms should be made accessible to all users. By using labels for each field, it's easy for users to identify what input is needed and easily navigate to that field. Any specific constraints or formatting should be clearly outlined, and required fields should not allow submitting if they're blank. Forms should not have a time limit, allowing each user to go at their own comfortable pace. Below is an example of an accessibly structured HTML form:

 
                    
    <form id="myform" method="post"> 
    <fieldset>
        <legend>Fill out this form.</legend>
        <label for="name">
            What is your name? *required
            <inout type="text" id="name" required>
        </label>
        <label for="your-email">
            What is your email? *required
            <inout type="email" id="your-email" placeholder="email@gmail.com" required>
        </label>
        <label for="phone">
            What is your phone number? *required
            <inout type="tel" id="phone" minlength="7">
        </label>
        <input type="submit" value="Submit">
    </fieldset>

Accessible Tables

When navigating tables with a screen-reader, it can be very easy to get lost and not know where in the table you are. In order to fix this issue, tables should indicate headers and their scopes, and separately define data cells. Including a caption will make it clear what the table content is about. Below is an example of an accessibly designed table:

 
                    
    <table> 
        <caption>Describe your table.</caption>
        <thead> 
            <tr> 
                <th scope="col">Column 1</th>
                <th scope="col">Column 2</th>
                <th scope="col">Column 3</th>
            </tr>
        </thead>
        <tbody> 
            <tr> 
                <td>Row 1, Column 1 data</td>
                <td>Row 1, Column 2 data</td>
                <td>Row 1, Column 3 data</td>
            </tr>
            <tr> 
                <td>Row 2, Column 1 data</td>
                <td>Row 2, Column 2 data</td>
                <td>Row 2, Column 3 data</td>
            </tr>
        </tbody>
    </table>

Mobile-First Development

With so many people accessing the web through their mobile devices, making your website accessible and usable on a mobile device should be a consideration from the beginning, not an afterthought. Beyond making sure website content is readable and formatted well across all size screens, certain functions like hover aren't visible on mobile devices, so it's crucial to not have that signal any important functionality. Making sure that navigation menus are clearly displayed in a format that is large enough to easily click and navigate on a phone screen for different mobility levels is another important consideration. Below is sample CSS code to make your horizontal navigation transform to a stacked, easily "thumb-able" format instead when the screen size is small:

 
                    
    @media screen and (max-width: 640px){
        nav ul{
            flex-flow: column nowrap;
            text-align: center;
        }
        
        nav ul li{
            margin: 0.3em 0;
        }

        nav ul li a{
            display: block;
        }
    }
