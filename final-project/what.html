What is Universal Design?

Universal design should meet the needs of as many people as possible. There can be many ways to approach universal design, and this page will help you narrow down the best ways to do it.

Web Standards

The W3C, founded in 1994 by Tim Berners-Lee, is an organization that seeks to standardize the protocols and technologies used by web designers in order to make the content available to as many people as possible. The W3C's goals prioritize accessibility, internationalization, privacy and security. Web standards can define the standard-compliant languages used for various website building purposes, such as the use of HTML for structure and CSS for style, to promote the development of web pages with syntax and code elements that can be recognized by a large range of browsers. Having web standards improves the accessibility of websites, as the uniform structure can be easier to navigate for those who use screen readers or keyboard navigation.

However, along with following these standardizations, there are also a few principles that you can follow to ensure that you're designing the best site you can.

7 Principles of Universal Design

The first set of principles we will go over are the 7 principles of universal design. These practices are applicable to both physical as well as the digital world, but for our purposes, they'll be described in context of the digital world.

The 7 Principles of Universal Design
Principle Description Example
Equitable Use The design should be usable to people with a diverse range of abilities, giving any visitor the opportunity to make full use of all the features. Using accessible design on a website, so all users can enjoy the features comfortably.
Flexibility in Use The design accommodates a wide range of individual user's preferences and abilities. Users can decide to read information or have it read out to them.
Simple and Intuitive Use User can easily determine how to navigate and make full use of website features without much prior specific knowledge, experience, or skills. Unneccesary complexity in design is eliminated.
Perceptible Information The design communicates all necessary information effectively to the user, regardless of the user's level of sensory abilities. Alternative texts on images mean that anyone can understand what is contained in the picture.
Tolerance for Error The design prevents the user from accidentally performing actions with unfavorable consequences if not intended. A website has the user confirm they want to log out when they click the log out button.
Low Physical Effort The design can be used comfortably without an excessive range of motion. Related website content is located in close proximity to each other so a user doesn't have to move their cursor too far.
Appropriate Size and Space for Approach and Use The design provides appropriate size and space for approach, reach, manipulation, and use, regardless of the user's body size, posture, or mobility. Website's buttons and fields are an appropriate size for any user to select, without having to be super precise in their movements.

Norman's Design Principles

Another set of general design principles that can be applied to make your website intuitive for all users is Donald Norman's design principles. Norman's concepts are useful to keep in mind for designing anything, and can help you to begin thinking like a designer, but some definitions must be extended for users who don't have certain sensory capabilities.

Norman's Design Principles
Principle Description Example
Visibility Elements should be clearly visible to a user, making it more likely the user can find it. Navigation menu items should not be hidden from the user's view, but should be clearly visible and found by a screen reader.
Feedback Every action should indicate to the user that it has been taken and what's been accomplished by it. Once a user successfully fill outs a field form, the form should indicate that the condition has been satisfied through color.
Constraints The interface should guide the user to a natural and appropriate next action. Form fields should be placed close together in a list to indicate the order to fill them out and show they are all related.
Mapping There is a clear relationship between different controls and their effect. The scroll bar on the side of a webpage moves down at the same pace as you're scrolling down on the page.
Consistency Elements and operations are similar for all elements that perform similar tasks. All buttons on a website are designed with the same style so the user can easily recognize that they should be clicked.
Affordance Attributes that allow people to know how to use it. Links have a line underneath to indicate the the word affords clicking.
